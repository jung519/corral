# 마실 사업 방향 피벗 정리

## 1. 사업 방향 피벗

기존 마실의 중심 질문은 다음에 가까웠다.

> 이 축제가 바가지 없이 믿을 만한가?

하지만 논의를 거치며 새롭게 정의한 중심 질문은 다음이다.

> 이 축제 재미있어?  
> 며칠 뒤 이런 재미있는 일이 열리는데, 내가 놓치고 있는 것은 아닐까?

따라서 마실은 단순히 지역축제 정보를 정리하거나 가격을 검증하는 서비스가 아니라, 곧 열릴 재미있는 행사, 축제, 팝업, 문화 이벤트를 미리 발견하고 사람들의 반응 신호를 기반으로 큐레이션하는 서비스로 피벗한다.

## 2. 새 사업 정의

마실은 공공 API에 등록된 지자체 축제만 다루지 않는다.

지자체 축제, 사기업 행사, 브랜드 팝업, 종교 행사, 대학 축제, 야시장, 플리마켓, 퍼레이드, 독립 문화행사까지 포함하는 검증된 경험 이벤트 DB를 구축한다.

### 기존 정의

> 축제 현장의 가격, 메뉴, 만족도를 검증하는 지역축제 큐레이션 앱

### 새 정의

> 곧 열릴 재미있는 축제와 이벤트를 사전에 발견하고, 사람들의 관심 신호와 경험 데이터를 바탕으로 지금 가볼 만한 것을 추천하는 큐레이션 서비스

## 3. 핵심 고객 문제

이제 문제는 단순히 축제 정보가 부족하다는 것이 아니다. 더 구체적으로는 다음 문제를 다룬다.

- 재미있는 행사가 열리는데 사용자는 사전에 알기 어렵다.
- SNS에서 뒤늦게 보고 "나만 몰랐네"라고 느끼는 경우가 많다.
- 공공 API에 없는 사기업, 종교, 팝업, 독립 행사는 발견하기 어렵다.
- 후기 기반 SNS 신호는 대부분 행사 종료 후에 강해져서 추천 시점으로는 늦을 수 있다.
- 사용자는 일정과 장소보다 먼저 "이게 나에게 재미있을지"를 알고 싶어 한다.

## 4. 데이터 전략

마실은 먼저 축제와 이벤트 정보를 사전에 확보해 DB에 저장한다.

그 다음 SNS, 검색, 영상, 뉴스, 사용자 반응 등에서 관심 신호가 올라오면 해당 행사를 전면에 큐레이션한다.

```text
사전 행사 수집
→ DB 등록 및 검증
→ 반응 신호 모니터링
→ FOMO/재미 점수화
→ 전면 큐레이션
```

### 주요 데이터 소스

- 공공 API: TourAPI, 문화포털, 지자체 문화행사 정보
- 행사 플랫폼: 이벤터스, 온오프믹스, 티켓/예매 플랫폼
- 팝업/전시 정보 서비스
- 네이버 블로그/뉴스/카페 검색 API
- 네이버 데이터랩 검색어 트렌드
- YouTube 검색/영상 데이터
- X 등 일부 공개 반응 데이터
- 공식 홈페이지, 공식 SNS, 보도자료
- 사용자 제보
- 주최자 직접 등록

Instagram과 TikTok은 FOMO 감정이 잘 드러나는 곳이지만, 직접 크롤링 기반 핵심 파이프라인으로 삼기에는 약관, 접근성, 안정성 리스크가 크다. 초기에는 참고, 수동 리서치, 제휴, 사용자 제보 중심으로 보는 편이 적절하다.

## 5. 장돌뱅이 재정의

### 기존 장돌뱅이

> 전국 축제 현장에 투입되어 가격과 메뉴 데이터를 수집하는 단기 고용 인력

### 새로운 장돌뱅이

> 반응은 높은데 정보 신뢰도가 낮은 행사에 선택적으로 투입되는 현장 검증 파트너

장돌뱅이는 더 이상 서비스 초기부터 비용을 쓰는 핵심 수집 인력이 아니다.

심사위원 관점에서는 매출 이전의 선행 비용 구조로 보일 수 있으므로, 사업기획서의 중심에서 내려야 한다.

### 새 역할

- 정보 충돌이 있는 행사 검증
- 반응이 급상승한 행사 현장 확인
- 공식 정보가 부족한 행사 확인
- 사진/영상 포인트, 현장 분위기, 실제 운영 여부 확인
- 크리에이터, 유튜버, 사용자 제보자까지 포함한 선택적 검증 네트워크

## 6. 운영 AI 재정의

### 기존 운영 AI

> 장돌뱅이가 올린 현장 데이터를 검수하고 승인, 기각, 정산하는 AI

### 새 운영 AI

> 행사 후보를 발견하고, 중복을 병합하고, 카테고리화하고, 반응 신호를 분석해 큐레이션 우선순위를 정하는 운영 코파일럿

### 주요 역할

- 공공 API 밖 행사 후보 발견
- 같은 행사의 다른 이름 병합
- 주최 유형 분류: 지자체, 사기업, 종교, 대학, 커뮤니티 등
- 경험 태그 추출: 야간, 퍼레이드, 사진맛집, 먹거리, 체험형, 데이트, 가족 등
- 검색량, 언급량, 영상 수, 뉴스 증가 추적
- 광고성, 홍보성, 중복 콘텐츠 감점
- 정보 신뢰도 산정
- 전면 큐레이션 후보 추천
- 현장 검증 필요 여부 판단

## 7. 개발 추가 필요 사항

개발해야 할 것은 단순히 크롤링과 AI 분석 2개가 아니다. 실제로는 다음 5개 영역이 필요하다.

| 구분 | 내용 | 유형 |
| --- | --- | --- |
| 행사 수집 파이프라인 | 공공 API, 검색 API, 행사 플랫폼, 공식 페이지에서 향후 열릴 행사 후보 수집 | 신규 |
| 행사 정규화/검증 시스템 | 중복 행사 병합, 날짜·장소·주최·공식 URL 검증, 출처 신뢰도 관리 | 신규 |
| 반응 신호 수집 파이프라인 | 네이버 검색/블로그/뉴스, 데이터랩, YouTube, X, 내부 클릭/저장 데이터 수집 | 신규 |
| AI 분석 엔진 | 행사 요약, 경험 태그, 카테고리, FOMO 신호, 광고성 판단, 큐레이션 근거 생성 | 신규 |
| 큐레이션 점수 시스템 | 일정 임박도, 관심도 상승률, 신뢰도, 경험 매력도, 사용자 관심도를 조합해 노출 순위 결정 | 신규 |

## 8. 기존 앱에서 개선할 부분

기존 CBT 앱을 완전히 버릴 필요는 없지만, 중심 기능은 바뀌어야 한다.

- 기존 축제 DB 스키마를 행사/이벤트 DB로 확장
- 지자체 축제 중심 구조에서 사기업, 종교, 팝업, 대학 행사까지 포함
- 가격/메뉴 중심 상세 화면을 경험 중심 상세 화면으로 재설계
- 만족도 신호등을 재미, 경험, 추천 대상, 현장감 중심으로 재정의
- 가격 정보는 핵심 가치가 아니라 보조 신뢰 정보로 이동
- 지도/캘린더 탐색 외에 "요즘 뜨는 행사", "이번 주 놓치면 아쉬운 행사" 큐레이션 추가
- 사용자 액션을 찜, 가고 싶어요, 다녀왔어요, 재밌었어요, 별로였어요 중심으로 재설계
- 운영자 검수 화면 추가
- 주최자 직접 등록/수정 기능 검토
- 장돌뱅이 기능은 상시 수집이 아니라 선택적 현장 검증 기능으로 축소

## 9. 새 마실의 한 줄 정리

> 마실은 곧 열릴 축제와 이벤트를 사전에 수집하고, 사람들의 관심 신호와 경험 데이터를 분석해 지금 놓치면 아쉬운 재미있는 일을 알려주는 큐레이션 서비스다.

이 피벗의 핵심은 가격 문제가 사라졌다는 것이 아니다.

가격, 메뉴, 바가지 정보의 위치가 바뀐 것이다. 이 정보들은 사용자를 끌어당기는 첫 이유가 아니라, 가고 싶어진 뒤 후회를 줄여주는 신뢰 정보로 내려간다.
